Core mental model (remember this):
Working Directory → Staging Area → Commit → Remote (GitHub)

*** the basics ***
A. Make changes
Edit files as usual

B. Open Source Control panel
You will see changed files
Click:
+ next to a file (stage one)
or + at the top (stage all)

C. Commit
Write a clear message (imperative tense):
Add user authentication logic
Click ✔️ Commit

D. Push to GitHub
Click Sync Changes or Push
Your commit is now on GitHub


***Branches (working safely)***

Create a branch:
Bottom-left branch name → Create new branch

Switch branches:
Click branch name → select branch

Merge branch into main:
Switch to main
Command Palette → Git: Merge Branch
Select your feature branch

*** Pull Requests (GitHub side) ***

Used when collaborating or reviewing work.
Typical flow:
Push branch to GitHub
GitHub shows “Compare & pull request”
Describe changes
Review → merge
VS Code can create PRs via:
Command Palette → GitHub: Create Pull Request

*** Pulling updates from others ***
Before starting work:
Click Pull or Sync Changes
Resolves differences between your copy and GitHub
If conflicts occur:
VS Code shows conflict markers
Choose:
“Accept Current”
“Accept Incoming”
or manual edit

*** Undoing mistakes ***
Discard uncommitted changes:
Source Control → right-click file → Discard Changes
Amend last commit:
Stage new changes
Commit → tick “Amend”
Revert a commit (safe):
Command Palette → Git: Revert Commit
